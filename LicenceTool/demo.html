📖 Implementierte Anforderungen

1️⃣ Unique Licence Keys

✓ Jede verkaufte Lizenz ist global eindeutig

Datenbank: UNIQUE Constraint + FOR UPDATE Locking

2️⃣ Multi-Item Purchase

✓ Benutzer können mehrere Produkte pro Bestellung kaufen

Schema: order_items mit quantity + assigned_keys Array

3️⃣ Role-Based Access

✓ Admin, Private User, Business User mit unterschiedlichen Rechten

Middleware: authorize() + checkResourceOwnership()

4️⃣ Admin Features

✓ Admin kann alle Bestellungen sehen und Refunds verarbeiten

Routes: /api/admin/orders (nur Admin)

5️⃣ Audit Trail

✓ Alle Operationen werden für Compliance geloggt

AuditService: logKeyAssignment(), logOrderRefunded(), etc.

6️⃣ Data Privacy

✓ Benutzer sehen nur ihre eigenen Daten

Middleware: checkResourceOwnership() prüft User-ID
