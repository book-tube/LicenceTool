📖 Implemented Requirements

1️⃣ Unique Licence Keys

✓ Every sold license is globally unique

Database: UNIQUE constraint + FOR UPDATE locking

2️⃣ Multi-Item Purchase

✓ Users can buy multiple products per order

Schema: order_items with quantity + assigned_keys array

3️⃣ Role-Based Access

✓ Admin, Private User, Business User with different permissions

Middleware: authorize() + checkResourceOwnership()

4️⃣ Admin Features

✓ Admin can view all orders and process refunds

Routes: /api/admin/orders (admin only)

5️⃣ Audit Trail

✓ All operations are logged for compliance

AuditService: logKeyAssignment(), logOrderRefunded(), etc.

6️⃣ Data Privacy

✓ Users only see their own data

Middleware: checkResourceOwnership() verifies user ID
